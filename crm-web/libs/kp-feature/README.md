# kp-feature

Фронтенд-модуль **коммерческого предложения (КП)**: конструктор, витрина каталога, шаблон печати.

This library was generated with [Nx](https://nx.dev).

## Конструктор КП (`KpBuilderPage`)

Позиции редактируются **только в превью документа** (таблица на макете A4). Кнопки удаления и поля ввода помечены классом `kp-no-print` и в `@media print` не попадают в PDF; в печати поля отображаются как текст.

Колонка **Фото** показывается только если хотя бы у одной строки заполнен `imageUrl` (например, из витрины при добавлении в КП); в ячейке только превью, в форме — скрытое поле. Размер миниатюры (px) под таблицей — после кнопки «+», только когда колонка «Фото» видна. В таблице сначала строки с фото, затем остальные.

## Витрина каталога (`KpCatalogVitrineComponent`)

Витрина показывает данные из справочника товаров (`GET /api/trade-goods`). Если API недоступен или данных нет — витрина остаётся пустой (без demo-данных). Товары из справочника приходят с **`imageUrl`** с бэкенда (`/media/trade-goods/…` из каталога `trade-goods-photos`); внешние placeholder-URL не используются.

Чтобы подставить данные в витрину извне, передайте массив в инпут **`[catalogProducts]`** (`KpCatalogProduct[]`). Пустой массив — пустая витрина.

См. также JSDoc на `@Input() catalogProducts` в `kp-catalog-vitrine.component.ts`.
