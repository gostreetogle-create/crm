Матрица разрешений: страницы, плитки на хабе «Справочники», кнопки в карточках и Excel. Роли — из справочника «Роли». Суперадминистратор в таблице не показывается: у него всегда полный доступ, матрица на него не действует. Для остальных: пока нет сохранённого переопределения для роли, действуют дефолты по коду роли из канона (например раздел «Справочники» и плитки хаба для director, viewer и т.д.). Как только вы сохранили набор галочек для роли — он становится единственным источником прав для этой роли (полная замена дефолта). Кнопка «Сбросить галочки» в колонке роли снимает все галочки для этой роли (явный пустой набор прав, без доступа к разделам и действиям). Порядок колонок — поле Порядок в «Роли». После входа матрица подтягивается с сервера (источник истины); в браузере — только кэш для быстрого UI. При выходе кэш сбрасывается. Изменения галочек — в кэш и (с правом page.admin.settings) на сервер (debounced).

Проверить интерфейс «как под другой ролью» в этом браузере — в Настройках (шестерёнка). Назначить роль сотруднику — в «Справочниках» → «Пользователи». Ключ страницы: page.admin.settings.
