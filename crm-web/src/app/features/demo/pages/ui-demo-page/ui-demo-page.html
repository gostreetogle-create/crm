Общая цепочка: dictionaryGrid → dictionaryTile → при необходимости app-hub-crud-expandable-shell → app-content-card → crud-layout. Ниже четыре варианта: что подключать решаете вы; отдельной «второй карточки» нет — это композиция тех же компонентов. Блоки с плитками вынесены из этой карточки, чтобы ширина колонки совпадала с /dictionaries (тот же dictionaryGrid внутри app-page-shell).

1. «Маленькая» плитка хаба — превью и раскрытие

app-hub-crud-expandable-shell + HubCrudExpandStateService (ключ плитки, здесь demo-etalon). В свёрнутом виде одна строка; после кнопки с тремя линиями — maxTableBodyRows снимается, tableBodyMaxHeight включает скролл. Раскрытый блок идёт поверх контента ниже (без сдвига страницы); при необходимости высоту резерва в потоке задайте входом layoutReserveWhenOpen на shell. Клик вне плитки сворачивает; кнопка с полосками внизу панели работает и в открытом виде. Так же на /dictionaries.

2. Эталон: полная карточка без кнопки раскрытия

Канон для «большой» карточки: app-content-card + crud-layout без app-hub-crud-expandable-shell, без maxTableBodyRows / tableBodyMaxHeight. В таблице — все поля модели строки (здесь: ID, название, категория, статус); на хабе в узкой плитке те же данные обычно сводят в одну колонку. Ширина — как у «широкой» плитки на хабе: dictionaryTileWide на всю сетку внутри той же контентной колонки, что и остальной текст (app-page-shell → .page), не на весь монитор.

3. Превью одной строки без раскрытия (редко)

Только входы CrudLayout: maxTableBodyRows="1", без app-hub-crud-expandable-shell. Лишние строки не видны и раскрыть нельзя — имеет смысл только для особых сценариев.

4. Полный список в узкой карточке

Как вариант 2, плюс maxInlineSize на app-content-card — карточка и тулбар не тянутся на всю ширину колонки грида (см. content-card.component.ts).
