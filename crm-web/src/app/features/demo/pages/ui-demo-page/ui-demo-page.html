Общая цепочка: dictionaryGrid → app-dictionary-hub-tile (при необходимости [fullWidth]="true" — на всю ширину сетки; shared/ui/cards/dictionary-hub-tile) → внутри те же app-hub-crud-expandable-shell → app-content-card → crud-layout. Ниже блоки-эталоны (в т.ч. 1.1 — широкая плитка с тем же раскрытием, что п. 1): что подключать решаете вы; отдельной «второй карточки» нет — это композиция тех же компонентов. Блоки с плитками вынесены из этой карточки, чтобы ширина колонки совпадала с /dictionaries (тот же dictionaryGrid внутри app-page-shell). Нумерация 1 / 1.1 / 2… — эталоны для хаба; 1.1 = тот же паттерн, что 1, с шириной как у 2.

Компонент app-dictionary-hub-tile (shared/ui/cards/dictionary-hub-tile): внутри app-hub-crud-expandable-shell + HubCrudExpandStateService (ключ demo-etalon). В свёрнутом виде одна строка; после кнопки с тремя линиями — maxTableBodyRows снимается, задаётся tableBodyMaxHeight (на узком экране — скролл внутри таблицы; на ширине ≥1024px тот же вход, но crud-layout снимает внутренний скролл — таблица по высоте целиком). Раскрытый блок идёт поверх контента ниже (без сдвига страницы); при необходимости высоту резерва в потоке задайте входом layoutReserveWhenOpen на shell. Клик вне плитки сворачивает; кнопка с полосками внизу панели работает и в открытом виде. Так же на /dictionaries (обычные справочники).

Тот же app-dictionary-hub-tile, что п. 1, с [fullWidth]="true": на хосте класс dictionaryHubTile--fullWidth → grid-column: 1 / -1 в dictionaryGrid (ширина как у п. 2). Ключ плитки demo-etalon-wide, чтобы раскрытие не смешивалось с п. 1. Для материалов и крупных обзоров на /dictionaries.

Канон для «большой» карточки: app-content-card + crud-layout без app-hub-crud-expandable-shell, без maxTableBodyRows / tableBodyMaxHeight. В таблице — все поля модели строки (здесь: ID, название, категория, статус); на хабе в узкой плитке те же данные обычно сводят в одну колонку. Ширина — как у «широкой» плитки на хабе: dictionaryTileWide на всю сетку внутри той же контентной колонки, что и остальной текст (app-page-shell → .page), не на весь монитор.

Только входы CrudLayout: maxTableBodyRows="1", без app-hub-crud-expandable-shell. Лишние строки не видны и раскрыть нельзя — имеет смысл только для особых сценариев.

Как вариант 2, плюс maxInlineSize на app-content-card — карточка и тулбар не тянутся на всю ширину колонки грида (см. shared/ui/cards/content-card/).
