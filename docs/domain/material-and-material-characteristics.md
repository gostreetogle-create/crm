# Материал и характеристики материала (целевая модель)

**Статус:** целевая модель зафиксирована здесь. На фронте (mock) тип **`MaterialItem`** и карточка «Материалы» на `/dictionaries` **приведены к разделению**: позиция ссылается на **характеристику** и **геометрию**; плотность/цвет/финиш/покрытие вводятся только в «Характеристиках материала». Excel и seed согласованы с этим.

## Смысл разделения

1. **Характеристики материала** — справочник **профилей/марок/типов** («алюминий», «сталь …», конкретная комбинация плотности, цвета RAL, финиша, покрытия и т.д.). Это **наследник по смыслу** нынешней перегруженной карточки «Материал»: всё, что описывает *что за вещество/покрытие*, а не *какую именно складскую позицию мы ведём*.

2. **Материал** — сущность **учётной строки**: ссылка на **характеристики**, ссылка на **форму и габариты** (существующий справочник геометрий), плюс минимальный набор полей, без которых нельзя закупку и остатки (цена за единицу, единица измерения, при необходимости внутренний код/название позиции).

3. **Форма и габариты** — как сейчас: отдельный справочник; материал **ссылается** на запись геометрии, а не дублирует параметры формы внутри характеристик (если позже понадобится «геометрия по умолчанию» для характеристики — обсуждается отдельно).

## Связи (логическая схема)

```
MaterialCharacteristic  (1) ──< (N) Material
         ↑
         │  опционально несколько материалов на одну характеристику
         │  (один профиль — разные геометрии / артикула)

Material  (N) ──> (1) Geometry   (форма и размеры)
Material  (N) ──> (1) Unit       (ед. изм., как сейчас)
```

Цвет / отделка / покрытие в целевой модели **живут в характеристиках** (или только FK + денорм для отображения/Excel), а не размножаются на каждую позицию «Материал», если она лишь сочетает «тот же профиль + другая геометрия».

## Карта полей: от текущего `MaterialItem` к целевому разнесению

| Поле (сейчас в материале) | Цель: **Характеристики материала** | Цель: **Материал** |
|---------------------------|-------------------------------------|----------------------|
| `name` | Имя профиля/марки (напр. тип стали/алюминия с отличиями) | Краткое имя **позиции** (SKU) или дублирующее отображаемое имя — по правилу учёта |
| `code` | Код марки/профиля (если есть) | Код **складской позиции** |
| `unitId` / `unitName` | Обычно нет (единица у позиции) | **Да** — цена «за 1 ед.» |
| `purchasePriceRub` | Нет (цена у позиции) | **Да** |
| `densityKgM3` | **Да** | Нет |
| `colorId` + `colorName`/`colorHex` | **Да** (FK + денорм) | Нет |
| `surfaceFinishId` + финиш-поля | **Да** | Нет |
| `coatingId` + покрытие-поля | **Да** | Нет |
| `notes` | Примечание к **профилю** | Примечание к **позиции** |
| `isActive` | **Да** | **Да** |

Итог: нынешняя модалка «Материалов» по полям ближе к будущей **«Характеристики материала»**; после введения второй сущности форма **Материала** сужается до выбора характеристики + геометрии + цены/единицы/кода/активности.

## UI-имена (рабочие)

| Сущность | RU в интерфейсе (предложение) |
|----------|----------------------------------|
| MaterialCharacteristic | **Характеристики материала** |
| Material | **Материалы** (учётные позиции) |
| Geometry | **Форма и габариты** (без переименования) |

Отдельный гайд-карточки в хабе: порядок размещения согласовать после появления второй сущности (типично: сначала опорные справочники, затем **характеристики**, затем **геометрия**, затем **материалы**).

## Этапы внедрения (чеклист)

1. Ввести тип/API **`MaterialCharacteristic`** (поля по таблице выше) и mock/репозиторий.
2. Упростить **`Material`**: FK на `characteristicId`, `geometryId`, перенос цены/единицы/кода/имени позиции.
3. Миграция данных: для каждой текущей строки материала — создать или найти характеристику, проставить ссылки (скрипт или ручной шаблон).
4. Хаб `/dictionaries`: новая плитка «Характеристики материала», переработка плитки «Материалы».
5. Обновить `crm-manufacturing-dictionaries.json`, `domainVsUi`, Excel-шаблоны, `costing-and-dictionary-prices.md` (цена остаётся на **позиции материала**).
6. Карточка **Деталь** (бэклог) будет ссылаться на **материал** и геометрию, а не раздувать справочник характеристик.
