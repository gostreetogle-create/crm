# Каталог: комплексы, товары и позиции (vs производство)

Краткая карта сущностей для продукта, аналитики и онбординга в код.

## Три уровня каталога (коммерческий контур)

| Уровень | Prisma-модель | Таблица БД | Смысл |
|---------|---------------|------------|--------|
| Комплекс | `Complex` | `complexes` | Группа каталожных товаров (например, жилой комплекс / проект). |
| Каталожный товар | `Product` | `products` | Товар внутри комплекса; имеет цену (`price` Decimal), связан с одним `complexId`. |
| Позиция / артикул в составе товара | `Article` | `articles` | Строка состава каталожного товара: наименование, количество (`qty`), порядок (`sortOrder`), связь с `productId`. |

Иерархия: **Complex → Product (каталог) → Article**.

---

## Не путать с производством и «Товарами»

| Что в UI / разговоре | Prisma | Таблица | Примечание |
|----------------------|--------|---------|------------|
| Плитка **«Изделия»** на `/справочники` | `ManufacturedProduct` | `"Product"` | BOM к **деталям производства** (`ProductLine` → `ProductionDetail`). |
| Плитки **«Каталожные товары»** | `Product` | `products` | Коммерческий товар в составе **комплекса**, не производственная BOM. |
| Плитка **«Товары»** (trade goods) | `TradeGood` | `trade_goods` | Коммерческая позиция из одного или нескольких **производственных изделий** (`ManufacturedProduct`). |

Итого: слово **«product»** в коде каталога (`crm-web` `CatalogProduct*`, `/api/catalog-products`) — это **каталожный** товар; **«products»** в API производства — **изделия** (`ManufacturedProduct`).
