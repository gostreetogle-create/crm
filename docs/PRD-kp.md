# PRD: КП (KP scenario)

## User stories
1. Пользователь хочет быстро выбрать товар и перейти к добавлению.
2. Пользователь хочет заполнить форму без лишнего скролла.

## Acceptance criteria
1. Форма выбора/добавления укладывается в первый экран на мобильном.
2. Кнопка `Добавить` всегда доступна (без необходимости скроллить вниз).
3. Ошибки в полях понятны и появляются рядом с полем после действия пользователя.

## Constraints
1. Мобильная версия поддерживается в рамках текущего UI-контракта проекта.
2. Использовать существующий подход к стилям (theme tokens / общие правила), без добавления новых зависимостей.
3. Реализовать в рамках текущих практик форм (например, Reactive Forms), чтобы не создавать разрыв между фичами.
