# PR checklist (обязательный)

Использовать перед сдачей каждой задачи.

- [ ] Использованы существующие `shared/ui` компоненты; если не хватало — сначала создан новый shared-компонент, потом применён в feature.
- [ ] В `features/*` нет локальных палитр/радиусов для типового UI; стили идут через theme tokens (`var(--...)`).
- [ ] Для форм соблюден единый UX: обязательные поля, момент показа ошибок, русский текст ошибок (или явно согласованное отключение текста), единый вид невалидного поля.
- [ ] Визуально проверено: при невалидном обязательном поле рамка поля красная (общий стиль через `UiFormFieldComponent` и токен `--field-invalid-border`).
- [ ] Если есть CRUD/диалог — поведение единообразно (`Esc`, `Отмена`, предсказуемое сохранение/сброс).
- [ ] Контракты/модели/репозитории синхронизированы с UI (нет “магических” полей только в шаблоне).
- [ ] `nx build crm-web` проходит без ошибок.
- [ ] По изменённым файлам нет новых lint-ошибок.
- [ ] Документация обновлена в том же изменении (`feature`-док + при необходимости `design-system`/`ui-reuse`/`workflow`).
- [ ] Если применялось временное локальное решение: есть запись в `docs/frontend/temporary-deviations-log.md` с `deadline` и `триггером исправления`.
