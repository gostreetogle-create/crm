# PR checklist (обязательный)

Использовать перед сдачей каждой задачи.

- [ ] Использованы существующие компоненты **`@srm/ui-kit`**; если не хватало — сначала добавлен примитив в `crm-web/libs/ui-kit`, экспорт в пакете, затем применён в feature.
- [ ] В `features/*` нет локальных палитр/радиусов для типового UI; стили идут через theme tokens (`var(--...)`).
- [ ] Для типовых действий (CRUD/filters/pagination/actions) использованы иконки `@lucide/angular` из shared-паттернов, без emoji и случайных SVG.
- [ ] Для иконок применена семантика цветов через токены темы (`--icon-affirm`, `--accent`, `--warning`, `--danger`, `--text-muted`), без локальных hex.
- [ ] Для форм соблюден единый UX: обязательные поля, момент показа ошибок, русский текст ошибок (или явно согласованное отключение текста), единый вид невалидного поля.
- [ ] Визуально проверено: при невалидном обязательном поле рамка поля красная (общий стиль через `UiFormFieldComponent` и токен `--field-invalid-border`).
- [ ] Если есть CRUD/диалог — поведение единообразно (`Esc`, `Отмена`, предсказуемое сохранение/сброс).
- [ ] Контракты/модели/репозитории синхронизированы с UI (нет “магических” полей только в шаблоне).
- [ ] `nx build crm-web` проходит без ошибок.
- [ ] По изменённым файлам нет новых lint-ошибок.
- [ ] Документация обновлена в том же изменении (`feature`-док + при необходимости `design-system`/`ui-reuse`/`workflow`).
- [ ] Если добавлялся новый theme token (в т.ч. для иконок), обновлены `crm-web/libs/theme-core/src/lib/theme-schema.ts`, `theme-presets.ts`, runtime в `theme.store.ts` и при необходимости JSON entry.
- [ ] Для входящих файлов из `analysis_hub` подготовлен mapping-спек: колонка источника -> поле UI-модели -> правило преобразования (и что делать с пустыми/грязными значениями).
- [ ] Если применялось временное локальное решение: есть запись в `docs/frontend/temporary-deviations-log.md` с `deadline` и `триггером исправления`.
- [ ] Для затронутого CRUD/карточного паттерна нет "точечных" расхождений между `/demo`, `/dictionaries` и соответствующими справочниками.
- [ ] Для справочников: нет дублирующего неподключённого CRUD-UI вне хаба `/dictionaries` (канон: `docs/frontend/dictionaries-crud-playbook.md`, «Порядок: единый хаб»).
- [ ] Если есть технически неизбежные исключения: добавлен checklist-долг (что не синхронизировано, почему, когда закрываем).
