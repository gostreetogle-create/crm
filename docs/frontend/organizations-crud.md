# Справочник «Организации» (для КП)

**Маршрут:** `/dictionaries`, плитка **Организации** (раздел «Клиенты»).

## Назначение

Карточка организации (**ООО** или **ИП**) для коммерческих предложений (КП), заказов и документов.  
Контактные лица выбираются из отдельного справочника **«Контактные лица»** (множественный выбор).

В API поле `legalForm` хранит строку **`ООО`** или **`ИП`** в соответствии с выбранным видом (свободного ввода ОПФ нет).

## Поля (UI / API)

| Группа | Поля |
|------|------|
| Карточка | **Вид организации** (ООО / ИП), Полное наименование, Короткое наименование |
| Реквизиты | ИНН (подсказка 10 или 12 цифр), **КПП** только для ООО, **ОГРН** или **ОГРНИП** (одно поле `ogrn`), ОКПО |
| Контакты | Телефон, Email, Сайт |
| Адреса | Юридический адрес, Почтовый адрес |
| Банк и подписант | Банк, БИК, Расчётный счёт, Корр. счёт, Подписант, Должность подписанта |
| Связи | Контактные лица (множественный выбор `contactIds`) |
| Дополнительно | Заметки, Активна |

## UX-правила для связей (канон для следующих таблиц)

Для связей на форме используем единый паттерн:

1. Поле выбора в виде выпадающего списка (`select`/`multiple`).
2. Рядом квадратная кнопка `+` для быстрого создания записи в связанном справочнике.
3. После создания запись должна появляться в списке выбора без перезагрузки страницы.

Этот же паттерн обязателен для следующих связанных сущностей КП (договоры, объекты, ответственные и т.д.).
