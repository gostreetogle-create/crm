# Ручной проход по ролям: справочники (бэклог #31–32)

## #31 — Видимость create по ролям

Для каждой роли (минимум `viewer`, `editor`, `admin` или ваши кастомные):

1. Войти под пользователем с ролью.
2. Открыть `/справочники`.
3. Проверить: плитки без права на просмотр не показываются; кнопки **+** / create скрыты или неактивны при отсутствии `crud.create` (или эквивалента в каталоге прав).
4. Для standalone-URL из [`dictionaries-canonical-paths.ts`](../../crm-web/libs/dictionaries-hub-feature/src/lib/dictionaries-canonical-paths.ts) при отсутствии прав — ожидание отказа guard (редирект/логин).

Канон прав: [`rbac-and-admin-settings.md`](./rbac-and-admin-settings.md).

## #32 — editor vs admin (роли и пользователи)

1. **Справочник «Роли»:** под `editor` — нет удаления системных ролей / ограничения по продукту; под `admin` — полный набор согласно каталогу.
2. **Справочник «Пользователи»:** создание/редактирование пароля и назначение роли — только при наличии прав; при необходимости отдельный сценарий «последний админ не удаляется».

Фиксировать расхождения в [`temporary-deviations-log.md`](./temporary-deviations-log.md).
