# Визуальное единообразие UI: чеклист (пункты 1–53)

Цель: проверить **каждый осмысленный пиксель** — сетку, типографику, контролы, состояния, адаптив. Поля ввода — пример; распространяется на весь интерфейс.

Канон: [`design-system.md`](./design-system.md), [`interaction-system.md`](./interaction-system.md). Отклонения — [`temporary-deviations-log.md`](./temporary-deviations-log.md). Справочники / плитки — правила hub в [`development-workflow.md`](./development-workflow.md).
Справочники CRM: [`dictionaries-crud-playbook.md`](./dictionaries-crud-playbook.md).

---

## A. Сетка и отступы (1–5)

1. **Модуль отступов** — только шаг из токенов (`--ui-space-*` и т.д.), без произвольных 7px / 13px.
2. **Поля страницы** — одинаковые отступы контента от краёв viewport на всех разделах одного класса (список / форма / хаб).
3. **Padding карточек** — одна «подушка» у карточек одного уровня (например `content-card`, плитки хаба).
4. **Вертикальное выравнивание** — левый край заголовков, полей первого уровня и основных кнопок на одной линии (или осознанный off-grid, задокументированный).
5. **Вертикальный ритм** — расстояния между секциями и между «заголовок → контент» одинаковы в однотипных экранах.

## B. Типографика (6–10)

6. **Иерархия** — h1 / h2 / подзаголовок / body / caption только из согласованных размеров и весов темы.
7. **Межстрочный интервал** — единый для подписей полей, хинтов и ячеек одного класса.
8. **Цвет текста** — primary / secondary / muted / disabled / placeholder из палитры токенов, не «почти как» с хардкодом.
9. **Длинный текст** — единая политика: `ellipsis` vs перенос vs многострочность в таблицах и карточках.
10. **Числа и единицы** — единый стиль для валют, %, мм (разделители, пробел перед единицей).

## C. Поля ввода и контролы (11–16)

11. **Высота** — одинаковая у input / select / primary-кнопок в одной форме (compact-канон из design-system).
12. **Скругление и обводка** — один радиус и толщина border для полей одного типа.
13. **Состояния** — default / hover / focus / error / disabled согласованы между всеми полями.
14. **Подпись поля** — одна схема (над полем / слева) внутри зоны продукта или явные зоны (модалка vs хаб).
15. **Обязательность** — маркер `*` или текст — единый стиль.
16. **Чекбоксы и переключатели** — выравнивание с подписью, один размер кликабельной зоны.

## D. Кнопки и ссылки (17–20)

17. **Варианты** — primary / secondary / soft / danger по interaction-system, не смешивать семантику.
18. **Размеры** — одинаковая высота кнопок в одном тулбаре; иконка + текст — единые зазоры.
19. **Состояния** — hover / active / disabled / loading без скачка вёрстки (ширина/высота ряда).
20. **Ссылки** — цвет, подчёркивание, hover — один канон.

## E. Иконки (21–23)

21. **Набор** — один источник (например Lucide), базовый размер строки действий согласован.
22. **Выравнивание** — иконка по центру с текстом в кнопке и в ячейке таблицы.
23. **Вторичные иконки** — не «плавать» по весу/контрасту между экранами.

## F. Карточки, панели, модалки (24–27)

24. **Тень и граница** — одна логика для карточки списка / модалки / выезжающей панели.
25. **Шапка карточки** — высота полосы, заголовок, действия справа — один паттерн.
26. **Модалка** — ширина, отступы контента, футер с действиями — согласованы между сущностями (`ui-modal`).
27. **Overlay** — затемнение и клик по фону — одно поведение.

## G. Таблицы и списки (28–32)

28. **Высота строки** — одинаковая в таблицах одного типа (например CrudLayout).
29. **Заголовки колонок** — шрифт, padding, иконка сортировки — единый вид.
30. **Разделители** — линии между строками: политика «есть / нет / только снизу» единая.
31. **Пустое состояние** — отступы и типографика как на других пустых экранах.
32. **Горизонтальный скролл** — если допускается, одинаковые подсказки (тень, fade).

## H. Навигация и шапка (33–35)

33. **Активный пункт** — подсветка навигации одного стиля.
34. **Высота баров** — top bar, хлебные крошки, вторичные полосы — согласованы по высоте и отступам.
35. **Мобильное меню** — отступы пунктов и разделители в том же ритме, что и десктоп-навигация по смыслу.

## I. Состояния данных и страницы (36–39)

36. **Загрузка** — скелетон vs спиннер: одна политика по приложению или явные зоны.
37. **Ошибка блока** — баннер / alert: один компонент и отступы от края контента.
38. **Успех** — toast / строка статуса: не три разных «зелёных» стиля без причины.
39. **Ошибки формы** — позиция сообщения (под полем / сверху формы) и длина строки — согласованы.

## J. Адаптив (40–42)

40. **Брейкпоинты** — проверка на типичных ширинах (например 375 / 768 / 1024 / 1440): нет обрезанных кнопок и висячих отступов.
41. **Тулбары** — перенос поиска и кнопок — одна логика на похожих экранах.
42. **Touch** — минимальная зона тапа для частых действий на мобильном (ориентир ~44px).

## K. Тёмная тема (43–44)

43. **Фоны слоёв** — карточка на фоне страницы: контраст без случайной «грязной» каймы.
44. **Границы** — в тёмной теме проверить видимость border и divider отдельно от светлой.

## L. Микровзаимодействия (45–46)

45. **Длительность** — hover, открытие модалки, раскрытие панели — из ограниченного набора длительностей.
46. **Focus ring** — видимый фокус для клавиатуры, не отключается «для красоты» без замены по WCAG.

## M. Как проверять (47–50)

47. **Эталон** — для спорного экрана: эталонный скрин или макет + текущий, сравнение рядом или overlay.
48. **Инспектор** — у расходящегося элемента: computed styles и источник правила (токен vs локальный override).
49. **Состояния** — не только happy path: ошибка, disabled, loading, пусто.
50. **Регресс** — после правки общего компонента пройти все экраны, где он используется.

## N. Как исправлять (51–53)

51. **Сначала токены и ui-kit** — потом фичи; иначе дубли и откаты.
52. **Одно отклонение — одна запись** в `temporary-deviations-log.md` со сроком/триггером снятия.
53. **Не плодить точечные костыли** — один источник правды на компонент; для справочников см. правила `CrudLayout` / hub в workflow.

## Быстрый поиск расхождений в коде

- Поиск «голых» контролов: `<input`, `<textarea`, `<select` вне общих обёрток в фичах.
- Поиск хардкода: цвета `#`, `rgb(`, произвольные `px` вне токенов в `features/` и `libs/*-feature` (по политике design-system).
