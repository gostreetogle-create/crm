# Field Contracts Canon

Цель: исключить расхождения между `form`, `payload`, `table`, `validation` в словарях и смежных CRUD-экранах.

## 1. Определение field contract

Field contract - это единый typed-источник, из которого однозначно выводятся:
- имя поля и тип;
- обязательность/валидация;
- отображение в форме;
- отображение в таблице;
- преобразование в payload.

## 2. Обязательные правила

1. Нельзя добавлять поле только в форму или только в таблицу.
2. Любое поле проходит цепочку:
   - модель формы;
   - payload builder;
   - table columns;
   - валидация;
   - тест/проверка.
3. Переименование поля делается atomically во всех точках контракта.
4. Значения enum/словарных ссылок не хранятся "магическими строками" в шаблонах.

## 3. Минимальный checklist на изменение поля

- Обновлен тип/контракт поля.
- Обновлены create/edit/view формы.
- Обновлен payload mapping.
- Обновлены колонки/formatters таблицы.
- Добавлен тест или fail-fast проверка на регрессию.
- Обновлены docs при изменении канона.

## 4. Антипаттерны

- Inline mapping в компоненте страницы без shared helper.
- Разные имена одного поля между UI и API без явного mapper.
- Временное расхождение без записи в `docs/frontend/temporary-deviations-log.md`.

## 5. Verification gates

- Локально: профильные unit/contract тесты + линт затронутой области.
- CI: fail-fast checks, затем `nx affected -t lint,test,build`.
- Review: `docs/frontend/pr-checklist.md` блок B обязателен.
