# Frontend Parity Policy (`crm-web` vs `srm-front`)

## Цель

Исключить функциональный дрейф между двумя Angular приложениями в одном монорепозитории.

## Обязательная parity-зона (P0)

1. `app.routes.ts`:
   - `/справочники` route,
   - child segments под `/справочники`,
   - legacy redirects на `/справочники`.
2. Guard/permission-логика для словарного хаба.
3. Контрактные smoke-тесты маршрутов.

## Правила изменений

- Любое изменение публичного route-контракта вносится в shared константы/фабрики.
- Ручные копии route-массивов между приложениями запрещены.
- Если изменение сделано только в одном приложении, PR считается неготовым.

## CI enforcement

- `check:route-parity` — fail-fast gate до запуска долгих Nx шагов.
- `nx affected -t lint,test,build` — основной пайплайн после fail-fast проверок.
- При отсутствии критичных workspace-файлов CI должен падать на `check:workspace-prereqs`.

## Исключения

- Временное расхождение допустимо только при записи в `temporary-deviations-log.md`:
  - срок закрытия,
  - причина,
  - критерий синхронизации.
