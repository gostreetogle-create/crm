# Video Docs Program

Формат: 4 обучающих видео для команды. Каждое видео заканчивается чеклистом "что проверить в PR".

## Видео 1 — Архитектура и границы

- Длительность: 12-15 минут
- Цель: показать, где source of truth и как не ломать Nx/route архитектуру.

### Покадровый план

1. Контекст репозитория и двух приложений (`crm-web` + `srm-front`).
2. Где живут shared route contracts и почему это важно.
3. Fail-fast проверки: workspace prereqs, route parity, mega-file guard.
4. Разбор типового drift-сценария и как его ловит CI.
5. Финальный чеклист для архитектора/ревьюера.

## Видео 2 — Словари и CRUD-контракты

- Длительность: 15-18 минут
- Цель: стандартизировать работу с `/справочники` и полевыми контрактами.

### Покадровый план

1. Канонический поток словарей и legacy redirects.
2. Shared CRUD-паттерн и зона доменной кастомизации.
3. Как синхронизировать form/table/payload для нового поля.
4. Антипаттерны: теневые страницы, точечные фиксы одной плитки.
5. Мини-демо: route parity test + CI gate.

## Видео 3 — PR-процесс и CI-гейты

- Длительность: 10-12 минут
- Цель: сделать PR-процесс повторяемым и проверяемым.

### Покадровый план

1. PR checklist: архитектура, field contracts, CI, docs.
2. Порядок прогонов локально и в CI.
3. Как читать FAIL от fail-fast скриптов.
4. Когда изменение считается PARTIAL vs DONE.
5. Шаблон финального отчета для PR.

## Видео 4 — Поддержка канона и техдолг

- Длительность: 12-14 минут
- Цель: научить команду не накапливать архитектурный шум.

### Покадровый план

1. Что такое architecture canon и parity policy.
2. Mega-file risk: когда и как декомпозировать.
3. Temporary deviations log: когда обязателен и как закрывать.
4. Ежемесячный mini-audit: что проверять в первую очередь.
5. KPI канона: предсказуемость, проверяемость, масштабируемость.
