# Temporary deviations log (обязательный при временных локальных решениях)

Цель: не терять временные локальные решения и фиксировать, когда их исправлять.

## Когда заполнять

Заполнять каждый раз, когда вносим временное локальное решение, которое не полностью соответствует shared-стандарту.

## Шаблон записи

```md
## [TD-YYYYMMDD-01] Короткое название
- **Дата:** YYYY-MM-DD
- **Где:** `path/to/file-or-feature`
- **Что временно:** кратко описать отклонение от стандарта
- **Почему временно:** почему нельзя сделать правильно сразу
- **Риск:** что может сломаться/разъехаться
- **Когда исправить (deadline):** дата или этап (например: "до подключения backend v0.3")
- **Триггер исправления:** событие, после которого обязательно исправить (например: "перед merge feature X")
- **План исправления:** 2-4 шага
- **Ответственный:** имя/роль
- **Статус:** open | done
```
