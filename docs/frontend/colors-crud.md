# Colors (RAL) CRUD

Цель: справочник `Цвета (RAL)` как единый источник выбора цвета для материалов.

## Где подключено

- Экран: `/dictionaries` (`src/app/features/dictionaries/pages/dictionaries-page/`).
- Провайдеры route: `src/app/app.routes.ts` (внутри `path: 'dictionaries'`).

## API-контракт (готов к backend)

- Модель: `src/app/features/colors/model/color-item.ts`
- Репозиторий: `src/app/features/colors/data/colors.repository.ts`
- Store: `src/app/features/colors/state/colors.store.ts`

Ключевые поля:

- `ralCode` (опционально, пример: `RAL 7035`)
- `name`
- `hex`, `rgb`

## Формат ввода (UI, единообразие)

- Для поля `Код RAL` в create-форме стартовое значение по умолчанию: `RAL `.
- Если пользователь вводит только числовую часть (например `7035`), frontend нормализует до `RAL 7035`.
- Если код заполнен, допустимы форматы:
  - классический: `RAL 0000` или `0000`,
  - `RAL DESIGN`: `RAL DESIGN 000 00 00` или `0000000` (нормализуется в `RAL DESIGN 000 00 00`).
- Если пользователь пытается стереть префикс, при blur поле нормализуется и не "теряет" `RAL`.
- Значение `RAL` без номера считается пустым (код не сохраняется).

## Интеграция с материалами

В форме материала:

- выбор цвета через dropdown из `ColorsStore.options()`,
- quick-add кнопка `+` открывает модалку создания цвета,
- после quick-add выбранный цвет автоматически подставляется в материал,
- `colorName` и `colorHex` синхронизируются из справочника,
- для backend отправляется `colorId` (если выбран).

## Массовый импорт (JSON / Admin API)

На плитке **нет** кнопок шаблона/импорта/экспорта. Массовое наполнение цветов — через будущий админский endpoint (формат и права — по аналогии с пилотом `POST /api/admin/bulk/units`, см. [`dev-bulk-json-migration-checklist.md`](../dev-bulk-json-migration-checklist.md)).

**Ориентир полей в payload** (эквивалент прежних колонок): `ralCode` (опционально), `name`, `hex` (`#RRGGBB`), `rgb` (`{ r, g, b }` 0…255). Правила валидации на сервере должны повторять смысл прежних проверок:

- обязательны осмысленные `name`, `hex`, согласованный `rgb`;
- `HEX` строго `#RRGGBB`;
- `RGB` строго в диапазоне и согласован с `hex`.

Ответ — список созданных записей и ошибок по индексам элементов, без «тихих» пропусков.
