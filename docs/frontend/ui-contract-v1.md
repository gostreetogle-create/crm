# UI Contract v1 (единый стандарт)

Цель: исключить расхождения поведения между страницами и закрепить переиспользование общего UI через **`@srm/ui-kit`**.

## 1) Source of truth

1. Временный эталон поведения: страница `/demo`.
2. Каноничные компоненты: **`crm-web/libs/ui-kit/`**, импорт в приложениях: **`import { … } from '@srm/ui-kit'`**.
3. Любой новый UI-паттерн сначала добавляется в **`libs/ui-kit`**, экспортируется из `libs/ui-kit/src/index.ts`, затем используется в feature-страницах.

## 2) Единые сценарии взаимодействия

1. `Редактировать`:
   - открывает единый сценарий редактирования;
   - кнопки действий: `Отмена`, `Сохранить изменения`.
2. `Создать`:
   - единый submit-флоу;
   - единый `Сбросить`.
3. `+` рядом с полем связи:
   - открывает quick-add окно;
   - после сохранения значение сразу подставляется в форму.
4. Удаление:
   - единый сценарий подтверждения (в следующей волне закрепляем отдельный confirm-pattern).
5. Закрытие модалки:
   - `Esc`, backdrop click, кнопка закрытия.

## 3) Единый фильтр-паттерн

Для списков/моделей используется одинаковый набор:

- Поиск (input),
- Сортировка (select),
- Фильтр по типу/категории (select).

Структура блока фильтров и naming должны быть одинаковыми на всех релевантных страницах.

## 4) Правила переиспользования

1. Запрещено дублировать локальные UI-реализации, если паттерн уже есть в **`@srm/ui-kit`**.
2. Если паттерна нет:
   - создаём новый компонент в **`crm-web/libs/ui-kit`**;
   - визуально и поведенчески подстраиваем под текущую тему и токены.
3. Feature-страницы содержат бизнес-логику, а не новые UI-примитивы.

## 4.1) Контракт по иконкам (обязательный)

1. Для типовых иконок интерфейса используем `@lucide/angular`.
2. Иконки в повторяемых сценариях (filters, row-actions, pagination, create/save/cancel/confirm) должны идти через компоненты **`@srm/ui-kit`** или согласованные паттерны из него.
3. Семантика цветов:
   - `--icon-affirm` — позитивные действия (`create/save/confirm`);
   - `--accent` — нейтральные действия/навигация/ссылки;
   - `--warning` — внимание/сроки;
   - `--danger` — опасные действия;
   - `--text-muted` — второстепенные элементы.
4. Запрещены emoji-иконки и произвольные inline-SVG для типовых сценариев, если есть эквивалент в Lucide.

## 5) Жизненный цикл demo-страницы

`/demo` — временный контрольный полигон.

Удаляем `/demo` только когда:

1. все ключевые паттерны закреплены в **`@srm/ui-kit`**;
2. аудит дубликатов закрыт;
3. behavior parity на feature-страницах подтверждена.
