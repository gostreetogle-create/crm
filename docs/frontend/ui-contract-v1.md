# UI Contract v1 (единый стандарт)

Цель: исключить расхождения поведения между страницами и закрепить переиспользование `shared/ui`.

## 1) Source of truth

1. Временный эталон поведения: страница `/demo`.
2. Каноничные компоненты: `crm-web/src/app/shared/ui/*`.
3. Любой новый UI-паттерн сначала добавляется в `shared/ui`, затем используется в feature-страницах.

## 2) Единые сценарии взаимодействия

1. `Редактировать`:
   - открывает единый сценарий редактирования;
   - кнопки действий: `Отмена`, `Сохранить изменения`.
2. `Создать`:
   - единый submit-флоу;
   - единый `Сбросить`.
3. `+` рядом с полем связи:
   - открывает quick-add окно;
   - после сохранения значение сразу подставляется в форму.
4. Удаление:
   - единый сценарий подтверждения (в следующей волне закрепляем отдельный confirm-pattern).
5. Закрытие модалки:
   - `Esc`, backdrop click, кнопка закрытия.

## 3) Единый фильтр-паттерн

Для списков/моделей используется одинаковый набор:

- Поиск (input),
- Сортировка (select),
- Фильтр по типу/категории (select).

Структура блока фильтров и naming должны быть одинаковыми на всех релевантных страницах.

## 4) Правила переиспользования

1. Запрещено дублировать локальные UI-реализации, если паттерн уже есть в `shared/ui`.
2. Если паттерна нет:
   - создаем новый компонент в `shared/ui`;
   - визуально и поведенчески подстраиваем под текущую тему и токены.
3. Feature-страницы содержат бизнес-логику, а не новые UI-примитивы.

## 5) Жизненный цикл demo-страницы

`/demo` — временный контрольный полигон.

Удаляем `/demo` только когда:

1. все ключевые паттерны закреплены в `shared/ui`;
2. аудит дубликатов закрыт;
3. behavior parity на feature-страницах подтверждена.
