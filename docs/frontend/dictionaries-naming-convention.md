# Dictionaries Naming Convention (RU, ГОСТ-style)

Цель: единый стандарт именования для всех текущих и будущих справочников в CRM.

## 1) Общий принцип

- У каждого справочника есть:
  - длинное наименование (для карточки/раздела и для **полей в модалке**),
  - короткое наименование (для **колонок таблицы** и плотных встроенных форм вроде материала).
- Длинные названия: понятные бизнес-пользователю.
- Короткие названия: краткие, однозначные, близкие по стилю к техдокументации/ГОСТ.
- Сокращения применяются только в UI-подписях, не в API-ключах и endpoint.

## 2) Каноничные пары (текущий список)

- `Единицы измерения` -> `Ед. изм.`
- `Тип отделки / шероховатость` -> `Тип отд.` / `Шерох.`
- `Форма и габаритные размеры` -> `Форма / габ. разм.` (если нужно сокращение в узких колонках)
- `Цвета (RAL)` -> `Название` (без сокращения для поля/колонки названия цвета)
- Короткий тег для вставок в другие справочники (в title модалки): `Цвет`
- `Вид работ` (карточка) -> тег в title модалки: `Вид раб.`
- Поля вида работ: `Короткое обозначение` -> в колонке `Коротк. обозн.`; ставка труда -> в колонке `₽/ч`, в модалке `Ставка, ₽/ч`

## 3) Правила сокращений (ближе к ГОСТ)

- Использовать точку в сокращениях: `Ед. изм.`, `Тип отд.`, `Шерох.`, `габ.`
- Не сокращать до неочевидных форм (`Тип отд` без точки, `Едизм`, `Шрх` и т.д.)
- Не смешивать 2+ варианта для одного термина в разных местах.
- Если термин спорный — сначала добавить пару "длинное/короткое" в этот документ, потом внедрять в UI.

## 4) Область применения

- Карточки справочников в `/dictionaries`.
- Заголовки таблиц (`CrudLayout` columns) — **короткие** подписи из п.2 (плотность).
- Формы в `UiModal` (create/edit/**view** одной сущности справочника) — **полные** подписи полей (как в п.2 «длинное» для термина), чтобы в большом окне не гадать по аббревиатурам.
- Встроенные блоки с множеством полей (например карточка материала) — допускаются **короткие** подписи по согласованным парам п.2, если иначе не помещается.
- Заголовки модалок в режиме просмотра: формат `Просмотр <длинное> — <короткое>`.
- Tooltip/aria-label для кнопок — использовать длинный понятный вариант.

## 5) Не распространяется на

- API DTO и JSON-поля (`name`, `code`, `isActive`, ...).
- Названия endpoint (`/units`, `/surface-finishes`, ...).

## 6) Definition of Done для новых справочников

- Добавлена пара "длинное/короткое" в этот файл (для каждого выводимого поля — по необходимости).
- В **колонках** таблицы — короткие подписи; в **модалке** справочника — полные подписи тех же полей (одни и те же TS/`key`, разные только строки label).
- Проверено отсутствие альтернативных сокращений в проекте (`rg` по ключевым словам).

## 7) Заголовки полноэкранного create (standalone, `/справочники/...`)

Источник строк: `crm-web/libs/dictionaries-hub-feature/src/lib/standalone-dictionary-create.meta.ts` (`title` у каждой строки `STANDALONE_DICTIONARY_CREATE`).

Правило: **длинное наименование** плитки на хабе (см. `dictionaries-hub-board.config.ts`, колонка `title`) и заголовок полноэкранной формы должны быть **согласованы по смыслу** — обычно формат «Новый/Новая …» + термин из п.2 (например `Новая единица измерения` ↔ карточка `Единицы измерения`; `Новый тип отделки / шероховатость` ↔ `Тип отделки / шероховатость`).

При смене названия карточки на хабе проверьте и при необходимости обновите `title` в meta для соответствующего `key`.
