# Materials CRUD (user-friendly screen)

Что реализовано:

1. Страница `/materials`:
   - список материалов,
   - создание материала,
   - редактирование материала,
   - удаление материала.

2. Данные пока берутся из mock-репозитория:
   - `crm-web/src/app/features/materials/data/materials.mock-repository.ts`
   - подключено через DI токен `MATERIALS_REPOSITORY`.

3. Страница:
   - `crm-web/src/app/features/materials/pages/materials-crud-page/`
4. UI-правила:
   - все кнопки через `UiButtonComponent` (`src/app/shared/ui/ui-button/`);
   - для обязательных полей выводим текст ошибки под полем (например, `name`).

## Для пользователя (простыми словами)

- Кнопка `Новый материал` — очищает форму для добавления.
- `Изменить` в строке — загружает материал в форму для правки.
- `Удалить` — удаляет строку из списка.
- При попытке сохранить пустое обязательное поле показывается понятная ошибка в UI.

## Дальше (когда будет backend)

- Добавить HTTP-репозиторий для материалов.
- Переключить DI с mock на http.
- UI останется тем же.
