# Materials CRUD (user-friendly screen)

Эталонный CRUD-справочник для последующего переноса остальных справочников в одну вкладку: повторяйте тот же каркас (`PageShell` → блоки `ContentCard` → `CrudLayout` + форма создания + `UiModal` для редактирования), тот же паттерн store/repository и те же UI-компоненты.

Что реализовано:

1. Страница `/dictionaries` (вкладка/блок «Материалы»):
   - список материалов,
   - создание материала,
   - редактирование в модальном окне,
   - удаление с подтверждением.

Примечание: маршрут `/materials` сейчас редиректит на `/dictionaries` (для обратной совместимости).

2. `CrudLayout`:
   - для эталона справочников используем «чистый» вид из Demo: только `title + table + actions` (без `subtitle` и `facts`, если это не отдельно согласовано),
   - индикация загрузки (пустой список не показывается как «Нет данных», пока идёт загрузка),
   - таблица на десктопе и карточки на узкой ширине.

3. Состояние и данные:
   - `MaterialsStore` подключается провайдером в `app.routes.ts` внутри route `/dictionaries` (не `providedIn: 'root'`), чтобы состояние было изолировано в рамках раздела справочников.
   - данные из mock-репозитория: `crm-web/src/app/features/materials/data/materials.mock-repository.ts`, токен `MATERIALS_REPOSITORY`.

4. Страница (единый UI):
   - блок «Материалы» на `crm-web/src/app/features/dictionaries/pages/dictionaries-page/` (модалка и форма; общий `ng-template` для create/edit).

5. UI-правила:
   - кнопки через `UiButtonComponent`;
   - поля через `UiFormFieldComponent` / `UiCheckboxFieldComponent`;
   - ошибка загрузки списка показывается над таблицей (`role="alert"`).
   - для ролевого доступа в шаблонах использовать `*appHasPermission`, а не разрозненные локальные флаги.

## Анти-дрифт правило

- Любые визуальные изменения `CrudLayout` сначала согласуются на `/demo` в секции «Плитка справочника: варианты (визуальный эталон)».
- После согласования изменение тем же PR переносится в `materials/geometries` (и далее в новые справочники).
- Нельзя оставлять ситуацию, когда Demo и справочники показывают разный базовый каркас таблицы.

## Для пользователя (простыми словами)

- Кнопка `Новый материал` — очищает форму для добавления.
- `Изменить` в строке — загружает материал в форму для правки.
- `Удалить` — удаляет строку из списка.
- При попытке сохранить пустое обязательное поле показывается понятная ошибка в UI.

## Модель «Материал» vs «Характеристики материала»

- **Характеристики материала** — отдельная плитка на хабе: плотность, ссылки/поля цвета, финиша, покрытия (как в доменном документе).
- **Материалы** — учётные позиции: название и код **позиции**, обязательные ссылки на **характеристику** и **геометрию**, **единица** и **цена закупки** за единицу, заметка, активность. Таблица и Excel показывают поля профиля **через join** (денорм только для отображения в `materialsData`).
- Импорт/экспорт материалов: в Excel для каждой связи из выпадающих списков — **id и подпись** (характеристика, геометрия, единица), плюс поля позиции. Импорт: обязательны **«Название»** и **«Цена ₽»**; характеристику ищем по **«ID характеристики»** или **«Код характеристики»**, геометрию по **«ID геометрии»** или **«Название геометрии»**, единицу по **«ID единицы»** или **«Код ЕИ»**. Подробнее: [`../domain/material-and-material-characteristics.md`](../domain/material-and-material-characteristics.md).
