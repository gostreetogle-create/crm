# Ручной прогон: полноэкранное создание (бэклог #10)

Цель: убедиться, что после **Сохранить** на каждом standalone-маршруте срабатывает возврат (`back`) и таблица на хабе показывает новую строку (store обновлён). Автотесты маршрутов и контракт URL не заменяют этот прогон.

**Предусловия:** пользователь с правом `page.dictionaries` и CRUD по нужной плитке; тестовая среда с API или моками.

## Общий сценарий (каждая строка таблицы)

1. Открыть хаб `/справочники`, выбрать плитку справочника (или перейти по прямой ссылке из колонки «URL»).
2. Заполнить форму **минимально валидно** (без дубликатов по уникальным полям).
3. Нажать **Создать** / **Сохранить**.
4. Ожидание: переход назад в истории (предыдущий экран), на хабе в таблице появилась новая запись (или после короткой задержки — polling store).

## Полноэкранные маршруты

| Справочник | URL (для deep link) | Минимальные поля (ориентир) |
|------------|---------------------|-----------------------------|
| Вид работ | `/справочники/новый-вид-работ` | Наименование |
| Единицы | `/справочники/новая-единица-измерения` | Код, название |
| Форма и габаритные размеры | `/справочники/новая-геометрия` | Название, форма |
| Цвет RAL | `/справочники/новый-цвет-ral` | Имя, hex, при необходимости код |
| Тип отделки / шероховатость | `/справочники/новая-отделка` | Тип, класс шероховатости |
| Покрытие | `/справочники/новое-покрытие` | Тип, спецификация |
| Организация | `/справочники/новая-организация` | Вид юрлица, наименование, ИНН (по правилам формы) |
| Контактное лицо | `/справочники/новый-контакт` | ФИО / обязательные поля формы |
| Роль | `/справочники/новая-роль` | Имя, порядок (если требуется) |
| Пользователь | `/справочники/новый-пользователь` | Логин, пароль, роль |

| Сценарий | URL | Примечание |
|----------|-----|------------|
| Новый материал | `/справочники/новый-материал` | После сохранения — ожидание строки в store по снимку полей, затем `back` |
| Новая характеристика | `/справочники/новая-характеристика-материала` | Аналогично; цепочка с материалом — см. playbook |

## Бэклог #11 (связь с кодом)

При **«Назад» без сохранения** с полноэкранной формы не должны залипать флаги quick-add (`unitQuickAddForMaterials`, `colorQuickAddForMaterialCharacteristics`, …). В коде: `resetAllDictionaryHubQuickAddFlags()` и закрытие вложенных модалок при выходе с маршрутов материала/характеристики.

Отметьте в `dictionaries-polish-backlog.md` п.10 после прохода по релизам или закройте задачу в трекере.

Цепочки материал / характеристика / propagation: [`dictionaries-regression-scenarios.md`](./dictionaries-regression-scenarios.md).
